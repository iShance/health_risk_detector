Algorithms Used
σ
Logistic Regression Stable
Outputs a probability 0–1 via the sigmoid function: σ(z) = 1/(1+e⁻ᶻ). The linear combination z includes all 6 features with learned weights plus interaction terms (BP×HR, Age×Chol). Best for linearly-separable patterns; fast and interpretable.
K
K-Nearest Neighbors Scale-sensitive
Finds the 7 most similar patients in training data using weighted Euclidean distance (closer neighbors vote more strongly). Features are normalised before computing distances. BP and Cholesterol carry higher weight (1.5×, 1.2×) reflecting clinical importance.
🌿
Decision Tree Interpretable
Creates human-readable rules: IF BP > 145 → Risk (94%), ELIF Cholesterol > 240 AND Age > 55 → Risk (75%), … Easiest to explain to clinicians. Can overfit on small datasets — depth is deliberately limited.
🌲
Random Forest Ensemble
An ensemble of 5 decision trees, each trained on a bootstrap sample with slightly different split thresholds. The final prediction is the average of all trees — this reduces variance and overfitting compared to a single tree, typically achieving the highest accuracy of the four models.

Limitations & Critical Thinking
Small synthetic dataset — not validated on real clinical data
Missing features: weight/BMI, family history, blood glucose, ECG readings
KNN is sensitive to feature scaling — normalisation is applied but synthetic data limits realism
Random Forest trees share the same feature set — true diversity requires bagged features (not implemented)
Never use as a replacement for professional medical diagnosis or treatment decisions
